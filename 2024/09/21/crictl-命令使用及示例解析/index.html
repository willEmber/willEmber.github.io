crictl 命令使用及示例解析

crictl 是什么？

crictl 是一个命令行工具，用于与 Kubernetes 节点上的容器运行时接口 (CRI) 进行交互。它允许你检查和管理节点上的 Pod、容器、镜像等资源。

1. 镜像管理

列出所有镜像

显示所有已下载的容器镜像。

拉取新镜像

从默认的镜像仓库下载最新的 nginx 镜像。

删除特定镜像

删除 nginx:latest 镜像。

删除所有未使用的镜像

删除所有未被容器使用的镜像，节省磁盘空间。

2. 容器管理

列出所有运行中的容器

只显示正在运行的容器。

列出所有容器（包括已停止的）

显示所有容器，包括已停止的。

启动一个已存在的容器

使用容器 ID 启动一个已创建但未运行的容器。

停止一个运行中的容器

优雅地停止一个正在运行的容器。

强制停止容器

立即停止容器，不等待优雅关闭。

删除一个容器

删除一个已停止的容器。

3. Pod 管理（主要用于 Kubernetes 环境）

列出所有 Pods

显示所有的 Pods。

创建一个新的 Pod

根据 JSON 配置文件创建一个新的 Pod。

停止一个 Pod

停止指定 ID 的 Pod。

删除一个 Pod

删除指定 ID 的 Pod。

4. 容器操作和调试

在运行中的容器内执行命令

在指定容器中交互式地执行 ‘ls /‘ 命令。

查看容器日志

显示指定容器的日志输出。

查看容器详细信息

显示容器的详细配置和状态信息。

查看容器资源使用统计

实时显示所有容器的 CPU、内存等资源使用情况。

5. 系统信息

查看 crictl 版本

显示 crictl 客户端和服务器的版本信息。

查看系统信息

显示容器运行时的详细系统信息。

使用技巧

  • 使用 crictl -h 或 crictl <command> -h 可以查看具体命令的帮助信息。
  • 大多数命令支持使用 -o json 选项来输出 JSON 格式的结果，方便进行进一步处理。
  • 在执行危险操作前，可以先使用 --dry-run 选项来模拟操作，查看会产生什么结果。
